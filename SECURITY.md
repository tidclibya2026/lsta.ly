# سياسة الأمن — LSTA

## الإصدارات المدعومة

الإصدار المدعوم حاليًا لأغراض التطوير والمراجعة هو:

| الإصدار | الحالة |
|---|---|
| Government Alpha 0.1.x | مدعوم |
| أقدم من 0.1.0 | غير مدعوم |

## الإبلاغ عن ثغرة

لا تنشر تفاصيل الثغرات أو الأسرار أو بيانات الوصول في GitHub Issues العامة.

أرسل البلاغ عبر القنوات الرسمية لمركز المعلومات والتوثيق السياحي، مع تضمين:

- وصف الثغرة.
- الوحدة المتأثرة.
- خطوات إعادة الإنتاج.
- مستوى الأثر المتوقع.
- أدلة غير حساسة.
- اقتراح المعالجة إن وجد.

## نطاقات حساسة

تُعامل العناصر التالية على أنها حساسة:

- بيانات الاعتماد ومفاتيح API وملفات `.env`.
- إعدادات قواعد البيانات والإنتاج.
- بيانات المستخدمين والأدوار.
- ملفات KML/KMZ والوسائط غير المنشورة.
- البيانات الحكومية قبل اعتمادها.
- نقاط النهاية الداخلية وصلاحيات المراجعين.
- سجلات التدقيق والنسخ الاحتياطية.

## قواعد أساسية

- لا تحفظ الأسرار داخل الكود أو Git history.
- استخدم أقل صلاحية ممكنة.
- راجع migrations قبل التطبيق.
- افصل بيئات التطوير وStaging والإنتاج.
- لا تنشر بيانات غير معتمدة للعامة.
- يجب أن تمر التغييرات الأمنية عبر Pull Request ومراجعة مستقلة.

## الاستجابة

تُقيّم البلاغات حسب الخطورة والأثر على السرية والسلامة والتوافر، ويُنسق الإصلاح والإفصاح وفق الإجراءات الحكومية المعتمدة.
