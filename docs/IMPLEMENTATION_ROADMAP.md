# خارطة تنفيذ أطلس ليبيا السياحي الذكي

## المرحلة 0 — الحماية والتأسيس

- تثبيت النسخة الحالية على `main`.
- اعتماد العمل عبر الفروع وPull Requests.
- إنشاء وثائق المشروع والحوكمة والمعمارية.
- إعداد سجل المخاطر والقرارات.

## المرحلة 1 — التدقيق والترحيل

- جرد ملفات GeoJSON وExcel والطبقات الحالية.
- توحيد الحقول والتصنيفات.
- فحص الإحداثيات والتكرارات والقيم المفقودة.
- إنشاء قاموس البيانات الوطني.
- إعداد قوائم التحقق الميداني.

**المخرج:** بيانات موحدة قابلة للتحميل إلى PostGIS.

## المرحلة 2 — قاعدة البيانات والخدمات

- إنشاء PostgreSQL/PostGIS.
- بناء الجداول والعلاقات والسجل التاريخي.
- تطوير API للمواقع والتصنيفات والمصادر.
- إضافة البحث والتصفية والتصدير.
- توثيق OpenAPI.

**المخرج:** نواة بيانات وطنية قابلة للتكامل.

## المرحلة 3 — الأطلس المؤسسي

- واجهة عربية وإنجليزية.
- خريطة وطنية متعددة الطبقات.
- بطاقة موقع موحدة.
- البحث المكاني والمرشحات والتجميع العنقودي.
- الطباعة والتصدير والمقارنة.
- بوابة إدارة الإدخال والمراجعة والاعتماد.

**المخرج:** نسخة تجريبية مؤسسية للأطلس.

## المرحلة 4 — دوائر التنمية والتنمية المكانية

- إدخال حدود ودراسات دوائر التنمية السابقة.
- ربط الدوائر بالمقومات والخدمات والبنية التحتية.
- تحليل الوصول والفجوات والمناطق المحرومة.
- تصنيف الأولويات التنموية.
- تحليل الملاءمة المكانية للمشروعات.

**المخرج:** بوابة التخطيط والتنمية المكانية.

## المرحلة 5 — الاستدامة والاستثمار

- مؤشرات الاستدامة البيئية والاجتماعية والاقتصادية والمؤسسية.
- الطاقة الاستيعابية والضغط السياحي وحساسية المواقع.
- مؤشر الجاهزية الاستثمارية.
- ملفات الفرص والمشروعات والسيناريوهات.

**المخرج:** لوحة استدامة وخارطة استثمار مترابطة.

## المرحلة 6 — الذكاء الاصطناعي

- فهرسة الدراسات والوثائق المعتمدة.
- بناء RAG مع pgvector.
- أوضاع المساعد: صانع القرار، المستثمر، الباحث، السائح، الموظف.
- تفسير المؤشرات والخرائط.
- اقتراح المسارات والفرص مع المصادر.

**المخرج:** مساعد افتراضي مؤسسي موثق المصدر.

## المرحلة 7 — Visit Libya والتشغيل الوطني

- API مشتركة للمواقع والصور والتصنيفات والمسارات.
- فصل البيانات المؤسسية عن المحتوى الترويجي.
- مزامنة محكومة مع Visit Libya وتطبيق الهاتف.
- اختبارات الأمن والأداء والإتاحة.
- تدريب المستخدمين والتشغيل التجريبي.
- الاعتماد والنشر الرسمي.

## أولويات أول 30 يومًا

1. تدقيق المستودع والملفات والطبقات.
2. اعتماد قاموس البيانات والتصنيفات.
3. إعداد مخطط PostGIS الأولي.
4. تجهيز حزمة استيراد طبقات الفنادق والقرى والمطاعم والمقاهي.
5. بناء نموذج بطاقة الموقع وحالة التحقق.
6. إعداد نموذج أولي لواجهة الخريطة الجديدة.
7. إعداد API أولية للتكامل مع Visit Libya.

## ضوابط التنفيذ

- لا تعديل مباشر على `main`.
- لا ترحيل بيانات قبل تقرير جودة.
- لا نشر دون مصدر وحالة اعتماد.
- لا دمج Pull Request دون مراجعة واختبارات.
- لا يستخدم المساعد الذكي بيانات غير موثقة في الإجابات الرسمية.
