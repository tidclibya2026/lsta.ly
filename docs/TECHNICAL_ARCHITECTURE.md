# المعمارية التقنية المستهدفة

## 1. المبادئ

- معمارية معيارية قابلة للتوسع.
- فصل الواجهة عن الخدمات وقاعدة البيانات.
- واجهات API موحدة وموثقة.
- الأمن والخصوصية منذ التصميم.
- قابلية التشغيل البيني مع الأنظمة الحكومية وVisit Libya.
- دعم العربية والإنجليزية وواجهات RTL/LTR.
- قابلية النقل بين بيئات التطوير والاختبار والإنتاج.

## 2. المكونات المقترحة

| الطبقة | التقنية المقترحة | الغرض |
|---|---|---|
| الواجهة | Next.js + TypeScript | البوابة العامة ولوحات المستخدمين |
| الخرائط | MapLibre GL JS | عرض وتحليل الطبقات المكانية |
| API | FastAPI | خدمات البيانات والتكامل والتحليل |
| قاعدة البيانات | PostgreSQL + PostGIS | التخزين العلاقي والمكاني |
| البحث | PostgreSQL FTS ثم OpenSearch | البحث النصي والدلالي |
| الذكاء الاصطناعي | خدمة Python مستقلة | RAG والتلخيص والتفسير والتوصيات |
| المتجهات | pgvector | فهرسة الوثائق والبحث الدلالي |
| الملفات | S3-compatible Object Storage | الصور والخرائط والوثائق |
| الهوية | Keycloak / OIDC | الدخول الموحد والصلاحيات |
| التشغيل | Docker | توحيد البيئات والنشر |
| الأتمتة | GitHub Actions | الاختبار والبناء والنشر |

## 3. الهيكل الوظيفي

```text
المستخدمون
  ├─ صانع القرار
  ├─ الموظف والمراجع
  ├─ المستثمر
  ├─ الباحث والطالب
  └─ السائح
        ↓
بوابات الويب والتطبيقات
        ↓
بوابة API موحدة
        ↓
خدمات المواقع | المؤشرات | الدراسات | الاستثمار | الذكاء الاصطناعي
        ↓
PostgreSQL/PostGIS | pgvector | التخزين الكائني
        ↓
Visit Libya والأنظمة الحكومية الشريكة
```

## 5. المعايير

- GeoJSON.
- OGC API Features.
- WMS/WMTS عند الحاجة.
- ISO 19115 للبيانات الوصفية الجغرافية.
- OpenAPI لتوثيق الخدمات.
- OAuth 2.0 / OpenID Connect.
- WCAG 2.2 AA لإتاحة الاستخدام.
- OWASP ASVS وOWASP Top 10 للأمن التطبيقي.

## 6. البيئات

- **Development:** التطوير اليومي والاختبارات الأولية.
- **Staging:** المراجعة المؤسسية والاختبارات التكاملية.
- **Production:** النسخة الحكومية المعتمدة.

لا يتم النشر إلى الإنتاج مباشرة من فرع ميزة، بل عبر Pull Request واختبارات واعتماد.

## 7. التكامل مع Visit Libya

- معرف وطني موحد لكل موقع.
- API للبيانات المعتمدة والصور والتصنيفات والمسارات.
- فصل الوصف المؤسسي عن الوصف الترويجي.
- مزامنة محكومة وقابلة للتتبع.
- روابط ثابتة مشتركة بين المنصتين.

## 8. الذكاء الاصطناعي

المساعد الافتراضي يعتمد على RAG ويشترط:

- استخدام الوثائق والبيانات المعتمدة فقط.
- إظهار المصدر وتاريخ التحديث.
- تسجيل الاستعلامات الحساسة لأغراض التدقيق.
- رفض الإجابة عند عدم توفر مصدر موثوق.
- فصل أوضاع الاستخدام: صانع قرار، مستثمر، باحث، سائح، موظف.

## 9. الانتقال من النسخة الحالية

تبقى خريطة Leaflet الحالية مستقرة على `main` إلى حين إنجاز النسخة الجديدة. يجري الانتقال عبر:

1. تدقيق البيانات الحالية.
2. تصميم المخطط الموحد.
3. ترحيل البيانات إلى PostGIS.
4. بناء API أولية.
5. تطوير واجهة الخريطة الجديدة.
6. اختبارات الأداء والجودة.
7. تشغيل تجريبي متوازٍ.
8. اعتماد النسخة المؤسسية.
