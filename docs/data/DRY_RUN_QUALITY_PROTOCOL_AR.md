# بروتوكول الفحص التجريبي وجودة بيانات أطلس ليبيا السياحي الذكي

**الجهة المالكة:** مركز المعلومات والتوثيق السياحي  
**الإصدار:** 1.0  
**حالة الوثيقة:** مرجعية تشغيلية قبل الاستيراد إلى قاعدة البيانات الوطنية

## 1. الغرض

يحدد هذا البروتوكول الإجراءات الإلزامية لفحص ملفات طبقات الأطلس قبل إدخالها إلى جداول `staging`، ثم قبل نقل السجلات المعتمدة إلى قاعدة الإنتاج أو نشرها إلى منصة Visit Libya.

## 2. نطاق الفحص الأول

يشمل الإصدار الأول الملفات التالية:

1. طبقة الفنادق.
2. طبقة القرى والمنتجعات السياحية.
3. طبقة المطاعم.
4. طبقة المقاهي.
5. الطبقات المنفصلة حسب المحاور.

## 3. خطوات التنفيذ

### 3.1 التحقق من الملف

- تثبيت اسم الملف وإصداره وتاريخ استلامه.
- حساب بصمة SHA-256.
- منع إعادة إدخال الملف نفسه دون قرار مسبب.
- تسجيل الجهة المالكة والمستخدم الذي قدم الملف.

### 3.2 الفحص البنيوي

- مطابقة اسم ورقة العمل مع ملف Mapping المعتمد.
- التحقق من وجود الحقول الأساسية.
- تسجيل الحقول الإضافية وعدم تجاهلها.
- عدم تعديل الملف الأصلي داخل مسار الاستيراد.

### 3.3 الفحص الموضوعي والمكاني

- وجود معرف ثابت للسجل.
- وجود الاسم الرسمي.
- صحة خط العرض وخط الطول ونظام الإسناد WGS84.
- وقوع الموقع داخل الحدود أو وجود مبرر موثق.
- إسناد البلدية ودائرة التنمية السياحية.
- وجود المصدر المرجعي وحالة التحقق.
- كشف الاشتباه في التكرار النصي والمكاني.

## 4. مستويات الخطورة

| المستوى | الوصف | قرار المعالجة |
|---|---|---|
| Critical | خلل يمنع الاعتماد المكاني أو المؤسسي | إيقاف السجل وإحالته إلى مراجعة GIS أو المصدر |
| Error | نقص جوهري في الهوية أو المصدر | عدم نقل السجل إلى الإنتاج |
| Warning | نقص يمكن استكماله مكتبيًا أو مكانيًا | يسمح ببقائه في staging فقط |
| Info | ملاحظة تحسين أو تنسيق | لا تمنع المراجعة |

## 5. بوابات الاعتماد

لا ينتقل السجل إلى الإنتاج إلا بعد اجتياز البوابات التالية:

1. **بوابة الهوية:** معرف واسم وتصنيف صحيح.
2. **بوابة المكان:** إحداثيات سليمة ووحدة إدارية محددة.
3. **بوابة المصدر:** مصدر ومالك بيانات وتاريخ تحقق.
4. **بوابة التكرار:** حسم حالات الدمج أو الفصل.
5. **بوابة النشر:** تحديد مستوى النشر المؤسسي أو العام.

## 6. قواعد Visit Libya

- الأطلس هو المصدر الوطني المرجعي.
- تستقبل Visit Libya السجلات المعتمدة والمعدة للنشر العام فقط.
- لا تنشر الملاحظات الداخلية أو درجات الشك أو بيانات الاتصال غير المصرح بها.
- يستخدم معرف وطني واحد في الأطلس وVisit Libya والتطبيق.

## 7. مخرجات كل دفعة

يجب إنتاج وحفظ:

- ملخص عدد السجلات.
- نسبة اكتمال الحقول الأساسية.
- عدد الأخطاء الحرجة والأخطاء والتحذيرات.
- قائمة السجلات التي تحتاج تحققًا ميدانيًا.
- قائمة حالات الاشتباه في التكرار.
- قرار الدفعة: مقبولة، مقبولة بشروط، مرفوضة، أو قيد المعالجة.
- اسم المراجع وتاريخ القرار.

## 8. المسؤوليات

| الدور | المسؤولية |
|---|---|
| مسؤول البيانات | استلام الملف وتسجيله وتشغيل الفحص |
| مختص GIS | مراجعة الإحداثيات والإسناد المكاني |
| مختص القطاع | مراجعة التصنيف والمعلومات السياحية |
| مسؤول الجودة | التحقق من تطبيق القواعد وتوثيق القرار |
| جهة الاعتماد | الموافقة على النقل إلى الإنتاج والنشر |

## 9. مبدأ عدم الفقد

يحتفظ النظام بالصف المصدري كاملًا، والنسخة المطبعّة، ونتائج الفحص، والقرار، وسجل التعديلات. لا يجوز استبدال السجل التاريخي أو حذف أثر المصدر عند التصحيح أو الدمج.
