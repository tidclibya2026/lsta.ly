# خطة ترحيل بيانات أطلس ليبيا السياحي وضمان الجودة

## النطاق الأول
تشمل الحزمة الأولى الملفات القطاعية التالية:
- الفنادق.
- القرى والمنتجعات السياحية.
- المطاعم.
- المقاهي.
- الطبقات المنفصلة حسب المحاور.

## قاعدة العمل
لا يتم إدخال ملفات Excel مباشرة إلى جداول الإنتاج. تمر البيانات عبر منطقة مرحلية `staging` ثم التحقق والتطبيع والمطابقة، وبعد اعتماد نتيجة الفحص تنتقل إلى الجداول الوطنية.

## خط الأنابيب
```text
Excel / GeoJSON / KML
        ↓
حفظ الملف الأصلي + checksum
        ↓
staging tables
        ↓
توحيد أسماء الحقول والقيم
        ↓
التحقق من الإحداثيات والتصنيف والبلدية
        ↓
كشف التكرارات النصية والمكانية
        ↓
قائمة مراجعة بشرية
        ↓
اعتماد السجل
        ↓
PostGIS production
        ↓
API الأطلس وVisit Libya
```

## مراحل التنفيذ
### 1. جرد المصدر
- اسم الملف وإصداره.
- الجهة المالكة.
- تاريخ الاستلام.
- عدد الأوراق والسجلات.
- checksum للنسخة الأصلية.

### 2. إعداد المطابقة
لكل ملف ينشأ ملف Mapping يربط:
- اسم العمود الأصلي.
- الحقل الوطني.
- قاعدة التحويل.
- القيمة الافتراضية المسموح بها.
- مستوى الثقة.

### 3. قواعد التحقق الأساسية
- وجود الاسم والمعرف والفئة والمصدر.
- صلاحية خط العرض وخط الطول.
- وقوع النقطة داخل الحدود الليبية أو وجود مبرر موثق.
- توافق الفئة الرئيسية والفرعية.
- إسناد البلدية بواسطة تقاطع مكاني.
- عدم نشر سجل بلا مصدر وحالة تحقق.

### 4. كشف التكرار
يحسب الاشتباه باستخدام:
- تشابه الاسم بعد إزالة الرموز والتطويل والمسافات الزائدة.
- المسافة بين الإحداثيات.
- الفئة والبلدية ورقم الهاتف والرابط.
- مصدر السجل.

لا يتم الدمج تلقائيًا عندما:
- تختلف الفئة.
- توجد تراخيص منفصلة.
- يكون الموقع فرعًا تابعًا لعلامة واحدة.
- توجد اختلافات زمنية تمثل تطويرًا أو توسعة للمرفق.

### 5. السجل التاريخي
يحفظ النظام:
- السجل قبل التعديل.
- السجل بعد التعديل.
- المستخدم المنفذ.
- سبب التعديل.
- تاريخ التنفيذ.
- المستند أو الدليل الداعم.

## حالات المراجعة
| الحالة | الاستخدام |
|---|---|
| مسودة | تم الاستيراد ولم يراجع |
| قيد المراجعة | يخضع للتدقيق المكتبي |
| تحقق مكتبي | اجتاز الفحص المكتبي |
| يحتاج تحقق ميداني | نقص أو تعارض لا يحسم مكتبيًا |
| معتمد | جاهز للاستخدام المؤسسي |
| مؤرشف | سجل تاريخي غير نشط |

## مؤشرات جودة البيانات
- نسبة اكتمال الحقول الأساسية.
- نسبة السجلات ذات مصدر موثق.
- نسبة السجلات ذات بلدية معتمدة.
- نسبة الإحداثيات الصحيحة.
- عدد حالات الاشتباه بالتكرار.
- متوسط عمر آخر تحقق.
- نسبة السجلات المعتمدة للنشر.

## ضوابط Visit Libya
- الأطلس هو المصدر المرجعي Master Data Source.
- Visit Libya لا تعدل الحقول المؤسسية.
- المحتوى الترويجي يعاد إلى الأطلس عبر دورة مراجعة.
- المزامنة تعتمد `site_id` الوطني.
- الصور لا تنشر دون توثيق الحقوق وحالة الاعتماد.

## مخرجات المرحلة الأولى
1. قاموس الحقول النهائي.
2. جداول staging لكل طبقة.
3. تقرير جودة لكل ملف.
4. قائمة التكرارات وحالات التحقق الميداني.
5. دفعة بيانات تجريبية معتمدة.
6. API تجريبية للقراءة فقط.
