# قاموس البيانات الوطني الأولي لأطلس ليبيا السياحي الذكي

## الغرض
توحيد الحقول المستخدمة في جميع طبقات الأطلس قبل ترحيل البيانات إلى PostgreSQL/PostGIS، مع ضمان قابلية الربط مع منصة Visit Libya، ولوحات دعم القرار، والمساعد الافتراضي.

## مبادئ إلزامية
1. معرف وطني فريد وثابت لكل موقع.
2. فصل الاسم الرسمي عن الاسم الترويجي.
3. اعتماد WGS84 / EPSG:4326 للإحداثيات الأصلية.
4. ربط كل سجل ببلدية، دائرة تنمية سياحية، ومصدر بيانات.
5. حفظ حالة التحقق والسجل التاريخي لكل تعديل.
6. عدم نشر أي سجل في Visit Libya قبل اعتماده مؤسسيًا.

- `draft`
- `under_review`
- `desk_verified`
- `field_verification_required`
- `approved`

## قواعد منع التكرار
1. مطابقة الاسم بعد التطبيع النصي.
2. فحص المسافة المكانية بين السجلات.
3. مقارنة البلدية والفئة والمصدر.
4. عدم الدمج الآلي إذا اختلفت الفئة أو الجهة المالكة.
5. توثيق قرار الدمج والسجل المتأثر.

## التكامل مع Visit Libya
لا ترسل المنصة إلا السجلات ذات الحالة `approved` وحالة النشر `visit_libya`، مع تمرير معرف الموقع الوطني بدل إنشاء معرف جديد في المنصة الترويجية.
